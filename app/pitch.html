Sojourn

A premade dev environment for JS apps.

Sojourn is a webpack toolchain which embraces a handful of tools, prebuilt to give you a quick way to jump into creating fantastic React-based software.

Clone it, annihilate its git history, and live code your way to ideation.

Modern tooling
Sojourn is a preconfigured webpack-based build system that mixes ES6, React, and Sass support together along with a joyful development experience.
Live code it
Sojourn is designed to be a relaxing design experience. It provides you with a development environment which keeps your React components and CSS hot-loaded in the browser. You shouldn't need to spam refresh all the time and rebuild app state to build effectively.
Build with
  • Live coding: All CSS and React component changes hot-load in your browser, so you can live code with minimal refreshing.
  • Babel: Use the cutting edge of Javascript with no hassle
  • React: Declare fast and stateless interfaces
  • Sass: CSS as it ought to be
  • Bourbon & Neat: Sensible defaults to get you styling fast
Test with
  • Mocha & Chai: A highly flexible testing interface so you can test just like you want. Although it starts with a describe/it/expect syntax, and a spec style suite runner, both tools are versatile and simple to configure.
  • Persistent specs with notifications: Keep your attention on your code and let your tests notify you when something goes wrong. Tests can be started and then backgrounded, and notifications will update you from time to time.
Get started now
  • git clone git@github.com:esmevane/sojourn
  • cd sojourn
  • npm install
  • rm -rf .git
  • npm run dev

This fires up the dev server, running by default on localhost:9001. Styling boilerplate is already there. Any changes you make to the code in the assets folder will persist while the dev server is running, and compile to app/assets.

Want to get your app ready for prime time? Just do this:

  • npm run build

Want to run the test suite? That's easy:

  • npm test

Want to keep the test suite running in the background, and rerun the suite every time you update it? That's easy, too:

  • npm run tdd
